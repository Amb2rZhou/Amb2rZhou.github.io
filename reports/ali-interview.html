三、Agent 知识备考（重点）

3.1 Agent 是什么（30 秒版）

Agent = LLM + 工具调用 + 记忆 + 规划能力。不是聊天机器人，是能自主完成多步骤任务的 AI 系统。

核心循环：感知 → 推理 → 行动 → 观察结果 → 继续（ReAct Loop）

3.2 Agent 核心组件

组件 作用 天猫超市场景举例
LLM（大脑） 理解意图、推理决策 理解用户”给我家猫买点吃的，要性价比高的”
Tool Use（手脚） 调用 API、查数据库、执行操作 调用商品搜索 API、比价接口、下单接口
Memory（记忆） 记住用户偏好、历史行为 记住用户养的是英短、之前买过某品牌
Planning（规划） 分解复杂任务为子步骤 “推荐猫粮”→ 确认猫种 → 筛选品牌 → 比价 → 推荐

3.3 Agent 架构流派（展示深度）

流派 代表 核心思路
Agent Loop 中心 Hermes Agent Agent 的执行循环是一切的起点，自动生成 skill
Gateway 中心 OpenClaw 消息路由和权限控制是核心，Agent 是组件之一
Harness 增强 intern-clawd 在现有 LLM 上叠加记忆+仪式+入口，不替换底层

你可以说：“我研究过主流 Agent 框架的架构差异——Hermes Agent 走 agent-loop-centric 路线，靠自动生成 skill 实现越用越聪明；OpenClaw 走 gateway-centric 路线，靠多平台覆盖建立护城河。我自己做的 intern-clawd 走的是第三条路：不做底层 agent，而是在 Claude Code 上叠加结构化记忆和仪式，让它从 coding 工具变成个人秘书。”

3.4 Agent × 天猫超市场景（必须准备）

场景 Agent 能做什么 PM 要考虑什么
智能导购 对话式购物：理解模糊需求 → 推荐 → 比较 → 下单 转化率、客单价、用户满意度
售后 Agent 自动处理退换货、物流查询、投诉分类 解决率、人工转接率、响应时间
商家运营 Agent 自动选品、定价建议、营销文案生成 商家 GMV、工具使用率、续费率
供应链 Agent 需求预测、库存优化、补货决策 缺货率、库存周转率、损耗率
搜索增强 自然语言搜索 → 意图理解 → 多轮澄清 → 精准结果 搜索转化率、零结果率

展开一个场景的示例回答：

“如果我来设计天猫超市的智能导购 Agent，我会分三步走。

第一步，意图理解：用户说’给我推荐适合 3 岁宝宝的零食’，Agent 需要理解年龄限制、安全标准、口味偏好。这里需要一个 domain-specific 的意图分类器，不是通用 LLM 直接回答。

第二步，工具调用：Agent 调用商品搜索 API（带儿童零食标签过滤）、用户历史购买记录（有没有过敏信息）、实时库存接口（确保能发货）。

第三步，个性化排序：基于用户画像（价格敏感度、品牌偏好、历史评价）做个性化排序，而不是按默认销量排。

核心指标我会关注：对话转化率（多少对话最终下单）、客单价提升（Agent 推荐 vs 自主搜索）、用户次日留存（Agent 导购体验是否让用户回来）。”

五、高频问题 & 参考回答

Q1: 你觉得 AI 会怎么改变电商？

“我认为 AI 对电商最大的改变不是效率提升，而是交互范式的重构。传统电商是’搜索→浏览→比较→下单’，AI 时代会变成’说一句话→Agent 帮你完成一切’。淘天已经在做这件事——AI 搜索理解语义需求，推荐从行为匹配转向兴趣推理。终局是用户不再需要逛，Agent 比你更懂你需要什么。”

Q2: 如果让你设计一个 Agent 产品，你怎么做？

用上面天猫超市智能导购的示例回答。

Q3: Agent 落地最大的挑战是什么？

“三个层面： 1. 可靠性——Agent 多步执行中一步错可能全盘崩，需要 human-in-the-loop 和 fallback 机制 2. 评估困难——不像搜索有明确的 NDCG，Agent 的好坏很难量化 3. 成本——每次对话消耗大量 token，需要在体验和成本之间找平衡（prompt caching、渐进式加载等）”

Q4: 你怎么理解产品经理这个角色？

“产品经理的核心职责是在用户价值和商业价值之间找到最优解。不是写 PRD 的人，而是定义’做什么、为什么做、怎么衡量做得好不好’的人。具体来说：发现真实需求（不是用户说什么就做什么）、定义优先级（资源永远不够，选最值得做的）、推动跨团队执行、用数据验证效果并迭代。”

六、面试前自检清单

七、Mock 面试题整理

Q1: 天猫超市最适合用 Agent 解决的问题是什么？

核心思路：天猫超市是高频、低决策成本、复购驱动（日用快消标品）。用户心智是”快速补货”不是”慢慢逛”。最适合管家型 Agent——记住购买周期、主动提醒、价格决策、一键下单。指标看月复购频次、Agent 触达后转化率、客单价。

Q2: Agent 的好坏怎么量化评估？

核心思路：分三层—— 1. 任务完成率（结果）：意图发起后有没有完成下单？类似搜索满足率 2. 过程质量（体验）：对话轮数、人工转接率、用户纠错次数 3. 商业指标（价值）：客单价、复购率、LTV

完成率判定靠隐式信号（下单/退货）+ 显式反馈（抽样 10% 弹👍👎）。早期人工标注建基准线，规模化后隐式为主。

Q3: Agent 在电商场景落地最大的风险？怎么控制？

核心思路：最大风险是自主决策出错导致信任崩塌。搜索给错用户划走就行，Agent 买错了是真金白银。

做三级权限：低风险全自动（查价格）→ 中风险确认后执行（下单）→ 高风险禁止自动（退款）。随信任积累逐步放权。原则：宁可多问一步，不可自作主张。

Q4: 你的情报系统能力怎么迁移到搜推广场景？

核心思路：情报系统的核心是从噪音中提取 signal。搜推广的核心问题一样——从用户模糊的搜索词中提取真实意图。

多源信号（搜索词 + 历史行为 + 用户画像）→ 意图推理 → 个性化结果。比如搜”健康零食”，结合用户上周买蛋白粉的行为 → 推蛋白棒而不是普通零食。

Q5: 加上 Agent 能力后，搜索推荐最大的变化是什么？

核心思路：从单轮交互变成多轮对话。搜索不再是独立模块，而是对话的第一轮。推荐不再是被动展示，而是 Agent 在对话中主动穿插关联推荐。

新增指标：对话深度（几轮完成购买）、跨品类渗透率（有没有带动关联购买）。

Q6: Agent 和 RPA 的本质区别？

核心思路：RPA 是”录制→回放”，流程固定、遇到意外就崩。Agent 是”给目标→自己想办法”，能理解、推理、应变。

类比：RPA = 流水线工人（按 SOP）、Agent = 实习生（给目标自己想办法）。不是替代关系，是各管各的层——确定性重复操作用 RPA，需要判断的用 Agent。

Q7: AI PM 和传统 PM 最大的区别？你为什么适合？

核心思路：最大区别是不确定性——传统产品行为确定，AI 产品行为概率性。

三个能力差异：评估方式（bad case 监控 vs 漏斗）、跟技术沟通（理解模型能力边界）、产品边界模糊（从意外用法发现机会）。

我适合因为我是 AI 产品的深度用户和构建者——搭过 Agent 体系、写过架构研报、每天在用在踩坑在迭代。

Q8: Agent 的记忆跟现有用户画像有什么区别？

核心思路：用户画像是结构化标签（行为数据被动采集，T+1 批量更新），Agent 记忆是非结构化上下文（对话中用户主动表达，实时生效）。

画像告诉你”该用户常买母婴产品”，Agent 记忆告诉你”宝宝 8 个月，刚开始吃辅食，对鸡蛋过敏”。两者互补，不是替代。Agent 获取的信息还可以回写到画像系统。

Q9: 入职第一天 leader 说”搜索结果页加个 AI 助手入口，你来负责”，第一周做什么？

核心思路：不急着画原型。Day 1-2 了解现状（跑一遍搜索链路、看数据看板、找”搜了但没买”的 case）。Day 3-4 定义问题（流失是因为”找不到”还是”选不了”？）。Day 5 输出 MVP 提案（先选一个垂直场景验证，定义 AB 对比指标）。

原则：第一周的产出不是方案，是问题定义。

Q10: 天猫超市 AI 助手和淘宝”问问”有什么区别？为什么不直接复用？（追问题）

核心思路：淘宝问问解决”不知道买什么”（探索性、多轮发散），天猫超市解决”帮我快点买完”（效率导向、越短越好）。

直接复用的问题：用户说”买牛奶”，淘宝问问会问”什么口味？全脂脱脂？“，但天猫超市用户期望的是”你上次买的 XX 牌还有货，比上次便宜 2 块，要下单吗？”

底层模型可以复用，但产品层的意图理解、对话策略、交互形态需要重新设计。

Q11: 天猫超市也在做非标品（生鲜、预制菜），Agent 策略跟标品有什么不同？

核心思路：非标品不确定性高（今天有什么、新不新鲜、口味合不合适），Agent 从”管家型”转向”顾问型”。

四个策略差异：①主动推荐替代品（缺货时不只说缺货）②时效感知（结合配送时间判断生鲜能不能送到）③场景化推荐（“今晚做什么菜”而不是”你该补货了”）④信任建设更难（需要更多解释降低决策焦虑）。

核心指标也不同：标品看复购率，非标品看品类渗透率和退货/差评率。

Q12: Agent 工具调用失败了怎么办？怎么设计容错？

核心思路：三层容错——

  1. 重试+降级：网络抖动自动重试；服务挂了用缓存降级，告知用户”信息可能有延迟”
  2. 局部失败不阻断整体：买牛奶成功纸巾缺货→不整体回滚，分别处理
  3. 诚实兜底：真搞不定→告诉用户转人工，带上对话上下文

原则：宁可坦诚说做不到，不可默默出错。可以结合 intern-clawd 的实践讲——Agent 遇到权限不够时告诉我原因和选项，不会自己硬上。

Q13: Agent 会不会取代传统搜推系统？两者什么关系？

核心思路：不会取代，分层共存。搜推是基座（大规模、高并发、低成本、100%用户），Agent 是增强层（复杂意图、多轮决策、高成本、10-20%用户）。

类比：搜推是超市货架（负责广度），Agent 是导购员（负责深度）。Agent 不应该是独立产品，而是嵌入现有搜推流程的增强能力——搜索结果页加”帮我挑”入口，推荐流加”为什么推这个”交互。

Q14: 搜推系统会被 Agent 取代吗？（深度版）

核心思路：短期分层共存，长期搜推退到后台变成 Agent 的”引擎”。用户不再直接跟搜推交互，而是跟 Agent 说话，Agent 在背后调用搜推能力。搜推从前台产品变成后台基础设施。淘天正在做的”从流量分发到理解系统”本质是在为 Agent 时代做基座准备。

Q15: Agent 时代搜推的新模式？

核心思路：从平台侧中心化推荐 → 用户侧去中心化推荐。传统搜推：画像存平台、算法是黑盒、数据归平台。Agent 时代：偏好存用户本地（MEMORY.md）、推荐逻辑透明可控、数据归用户。推荐权力从平台转移到用户。

对天猫超市既是机会（率先把搜推能力开放给用户 Agent）也是威胁（用户 Agent 跨平台比价，平台锁定力下降）。

Q16: 你的项目都是一个人做的，怎么证明跨团队协作能力？

核心思路：用实际经历回答，不要吹大—— 1. 跟端到端 coding agent PM 配合做技术选型和约外部访谈 2. 情报系统服务技术高管，根据反馈加了 benchmark collector、把 webhook 文字改成文档形式（卡在 API 权限，后来用内部 Lobi 实现） 3. API 费自己垫：先用最小成本验证产品价值再争取资源，这是 PM 思维

Q17: 情报系统的 LLM 筛选逻辑？怎么保证不漏重要信息？

核心思路：不是让 LLM “凭感觉选”，prompt 里定义了明确框架——8 个视角维度、signal 强度判断标准（first/影响面/时效性/信源权威度）、溯源规则。经过 4 轮调优，本质是用 prompt 编码的评分函数。

保证不漏靠多源交叉覆盖 + prompt 迭代。但坦诚说 10 条一定会漏——选择了高 precision 而非高 recall，因为用户是忙碌的决策者。

Q18: 反问面试官

推荐问 2 个： 1. “天猫超市目前在 Agent 方向最大的技术或产品卡点是什么？” 2. “这个岗位入职后前三个月，你期望实习生能交付什么？”

Q19: GMV 突然下降 30%，怎么拆解原因？

核心思路：先确认数据真实性（口径问题？大促基数？bug？）→ 拆 GMV = 用户数 × 转化率 × 客单价，定位哪个因子降了 → 追问为什么只有这个城市（天气？仓储？竞对补贴？政策？）。

回答框架：先说”我的分析思路是沿 GMV 公式逐层拆解，同时交叉验证是全局还是局部问题”，然后让面试官选方向深入。

Q20: 怎么设计 AB 测试验证 AI 推荐效果？

核心思路：五步—— 1. 定义假设（AI 推荐能提升转化率） 2. 按用户 ID 随机分组，先 5% 小流量再放大 3. 指标三层：主指标一个（转化率）、辅助指标（客单价/加购率）、护栏指标（退货率/客诉率不能变差） 4. 样本量提前算好，至少跑 7 天，避开大促 5. 注意坑：新鲜感效应（看第 2 周不只看第 1 周）、辛普森悖论（分层看）、多重检验（主指标只定一个）

Q21: 上线前怎么做离线评估？

核心思路：三层—— 1. 自动化指标：用历史数据回测，Recall@K（推的准不准）、NDCG（排序好不好）、覆盖率（是不是千人千面） 2. 人工 case review：抽 100-200 case 标注合理/不合理/有风险。天猫超市 bad case：推过敏食品、推刚买过的、推缺货的、推不符合家庭结构的 3. Agent 特殊评估：用历史对话 log 回放，LLM-as-judge 打分

离线保证下限，在线验证上限。

Q23: 新鲜感效应消退 vs 产品本身有问题，怎么区分？

核心思路：看两组数据——①按用户分群：第二周新进的用户数据好不好？好→产品没问题，老用户只是新鲜感过了。②看使用深度不只看频次：留存用户对话轮数更多、下单率更高→说明留下的是真实需求用户。判断标准：第二周留存用户人均 GMV 没降甚至涨了 = 健康消退。

Q24: AI 搜索怎么处理”好喝的牛奶”这种主观搜索词？

核心思路：两层——①群体共识：用评价数据挖掘哪些牛奶”好喝”出现频率高。②个人偏好：结合用户历史（一直买全脂→“好喝”对他是全脂）。新用户做一轮轻量澄清。主观词靠数据翻译成客观特征。

Q25: 怎么验证”兴趣推理”真的比”行为相似”好？

核心思路：AB 测试看两个指标——①点击率 ②惊喜度（推的商品里有多少是用户没浏览过的品类但最终买了）。惊喜度高且转化不差 = 兴趣推理真的理解了用户。淘天数据：推荐点击+10%、购买效率+25%。

行为相似 vs 兴趣推理区别：行为相似是”买A的人也买了B”（统计关联，不知道为什么）。兴趣推理是”你买了奶粉+看了辅食教程+搜了8个月宝宝 → 你家有8个月宝宝在过渡到辅食 → 推米粉磨牙棒”（语义理解，知道为什么）。

Q26: AI 广告动态出价，商家怎么信任你？

核心思路：①透明度——展示 AI 做了什么决策。②保底机制——商家设 ROI 下限，低于自动停投。③渐进信任——小预算先跑一周拿数据，再加预算。个性化效果报告比平均值重要。

Q27: AI 美工 2 亿张图怎么保证质量？

核心思路：自动化门控100%覆盖（品牌一致性/违规检测/基础质量）+ 人工抽样1-5%（重点审新品牌新品类）+ 商家兜底（预览修改/一键回滚）。自动化保下限，人工保上限，给商家最终决定权。

Q28: AI 客服省 2000 万但”听不懂”，怎么平衡成本和体验？

核心思路：分层服务——简单问题全自动（查物流/退换货政策，准确率95%+），复杂问题及时转人工（带上下文）。关键指标不是”AI拦截率”而是首次解决率、转人工后满意度、重复来电率。省钱不是靠堵住转人工的路，是靠让 AI 解决它擅长的 80%。

Q29: AI 帮我挑 vs 搜索的本质区别？产品形态？

核心思路：搜索=用户主导决策（AI 只负责找），帮我挑=用户让渡决策权（AI 负责找+选）。产品设计：入口放搜索结果页顶部、最多2-3轮对话、输出2-3个选项附推荐理由、直接可加购不跳详情页。

Q30: AI 试穿退货率反而上升了怎么办？

核心思路：核心矛盾是效果太好=期望过高=退货上升。解决方向是管理预期：①真实感优先（保留不完美）②加买家秀对比 ③标注模拟置信度（纯色T恤=高，蕾丝=低）④数据闭环追踪试穿后购买且未退货率。核心指标不是使用率，是试穿后购买且未退货率。

Q31: 智能购物车凑单功能的成功指标？

核心思路：目标是双赢——用户省钱，平台提客单价。用户侧看凑单点击率、实际省了多少、购物车放弃率变化。平台侧看客单价提升、凑单商品毛利率、订单完成率。护栏看退货率和凑单后取消率。

主指标：凑单带来的增量 GMV ×（1 - 凑单商品退货率）。乘退货率是防虚假繁荣。

Q32: 你选模型的标准是什么？不同场景怎么选？

核心思路：四个维度按优先级——任务匹配度（自己跑 case 看效果，不看 benchmark）、成本、延迟、稳定性。

情报系统选 DeepSeek（离线、低量、性价比高）。AI 客服选小模型处理简单问题（快+便宜）+ 大模型兜底复杂问题（准+贵）。不是所有问题都需要 Opus 来回答”快递到哪了”。

Q33: AI PM 需要会写代码吗？技术背景是优势还是包袱？

核心思路：三个优势——①需求可行性直接判断不用等开发回复 ②跟工程师沟通零翻译成本 ③能自己写原型快速验证。包袱：容易陷入技术细节忘了用户视角（上轮面试官”思路打开”的反馈就是这个）。

一句话：技术是看问题的放大镜，但决策的准心应该始终对准用户价值。

Q34: 面试备用——你怎么看 MCP（Model Context Protocol）对电商的影响？

核心思路：MCP 让 Agent 可以标准化地接入任何服务。对电商意味着：Agent 不再局限于一个平台，可以同时调天猫、京东、盒马的接口帮用户比价下单。

对天猫超市的威胁：如果用户的 Agent 可以跨平台，那平台的竞争从”谁的 app 体验好”变成”谁的 MCP 接口更开放、数据更丰富”。防守策略是用 Context 锁住用户——你的购买记录、偏好、地址都在我这里，迁移成本高。
