6b.1 大模型语料采集 6 题（蚂蚁风格的 AI 专业题）

Q. 大模型语料采集和清洗应该注意什么？
  • 采集维度：多样性（避免偏差）/ 时效性 / 合规性（版权 / 隐私）/ 质量分层
  • 清洗维度：去重（fuzzy match）/ 噪音过滤（乱码 / 广告 / 低质 UGC）/ 格式标准化 / 毒性/违规内容过滤
  • 金融场景特殊：PII 脱敏（身份证 / 账号 / 金额）+ 合规关键词过滤 + 引用溯源
Q. 如何评判大模型语料采集的质量？

分四维度：

  1. 完整性：是否覆盖 target domain 的关键子域（可用 topic modeling 检测 gap）
  2. 准确性：事实一致性（人工 spot check + LLM-as-judge 批量评估）
  3. 多样性：语言风格 / 观点立场 / 时间跨度
  4. 时效性：数据新鲜度分布

举具体例子（如果面试官追问）：金融语料评判——"好"的金融语料应该覆盖研报 / 新闻 / 公告 / 行业分析 / 监管文件等 5 类来源，缺任何一类都有 gap。

Q. 如何计算大模型语料采集的完整性？
  • 先定义领域分类 schema：比如金融分 18 个子域（权益 / 固收 / 量化 / 合规...）
  • 采样统计：每个子域的样本占比 vs 期望占比（用 chi-square 检测分布偏差）
  • embedding 空间覆盖率：把语料和"理想全集"都 embed，看 embedding cluster 覆盖率
  • 外部验证：用 held-out 问题集（未见过的 query），看模型能不能答对 → 答不对的说明语料覆盖不够
Q. 大模型语料采集的难点是什么？
  • 高质量语料获取成本高：专业内容（法律/金融/医疗）都有版权，爬虫违规风险
  • 合规和可用性的平衡：脱敏后语料的语义损失 vs 合规刚性要求
  • 长尾场景难覆盖：通用语料好找，专业细分场景（如"东南亚伊斯兰金融合规"）找不到
  • 质量评估主观性：没有 ground truth，评估依赖人工或 LLM-as-judge，成本高
  • 语料时效 decay：金融语料半年前可能已过时，需要持续更新但成本高

6b.2 Boss 的 AFI 挂钩点

这些大模型语料题你可以用 AFI 的经验回答——AFI 本质就是做多源信号采集 + LLM 相关性打分：
  • 采集：RSS / GitHub / ArXiv / HuggingFace 多源（vs 大模型多源）
  • 打分：LLM 按相关性筛选（vs 大模型质量评估）
  • 去重：我处理过 signal 的去重（vs 大模型语料去重）
  • 时效：AFI 的 daily 机制（vs 语料更新机制）
"我做 AFI 的时候在小体量上踩过这些坑，对大模型语料采集的难点有直观体感。"
