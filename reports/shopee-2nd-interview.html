🔥 三、3 个必问题 · 精准回答

Q1. 为什么来 Shopee？（必问）
公式：具体的 3 个理由，每个带证据
  1. 东南亚是 AI 搜索 hard mode 的试金石——一面面试官告诉我东南亚有三个特有约束（多语言、低 AI 素养、低 query 质量），这三个问题解决了的产品能力是可迁移的全球资产。国内做优化，Shopee 做破局。
  2. Shopee 的 AI 策略是整体的，不是孤岛——我注意到你们做 toC 导购、内部 Copilot、和 ChatGPT MCP 三个层次，这是一个面向 Agent 时代的完整布局。我的 intern-clawd 项目就是个人级的这个思路。
  3. Q1 2026 GMV 52% 市场份额说明 Shopee 在东南亚是行业领导者——这种规模的数据和基础设施是做 AI 搜索实验的最佳场景。
不要说：互联网太卷 / 想去东南亚 / 喜欢跨境 —— 这些都太虚。要给具体的"我看到 X"。
Q2. 你对 Shopee 了解多少？（必问）
公式：市场 × AI × 竞争 三个维度
  • 市场：Q1 2026 668 亿美金 GMV，占东南亚电商 52%；印尼 44%（主战场）/ 泰国 YoY 22% / 马来 YoY 19%；东南亚总市场预计 2026 底 2150-2300 亿美金
  • AI 能力：
    • 搜索已用 NLP + 深度学习理解语义
    • Semantic search + voice + image search
    • AI Listing Optimizer（给商家的 AI 工具，自动生成标题 / 描述 / 标签）
    • 区域化个性化（Hari Raya in Malaysia, Ramadan in Indonesia 动态分群）
  • 竞争：
    • TikTok Shop 崛起（最大威胁）
    • Lazada（传统竞对，市场份额被持续压缩）
    • 护城河：integrated logistics + fintech (SPayLater)
讲 3-4 个关键数字 + 一个战略判断，不要都讲完。让面试官追问你"你觉得 TikTok Shop 是最大威胁吗"之类。
Q3. 一面你从我们这里学到了什么？（可能问）
  • "一面最打动我的是你们的三层职能——不是把 AI 搜索当成一个功能，是把它当成一个生态"
  • "toC 导购服务消费者，内部员工 Copilot 服务运营效率，ChatGPT MCP 对接外部 Agent——这三层对应了从 'AI in product' 到 'AI as infrastructure' 到 'AI as gateway' 的完整谱系"
  • "我回去查了一下，发现很多大厂（阿里、字节）都在做第一层，很少有人把三层都跑通的——这也是我想加入的原因"
这题背后的问题：面试官在测你会不会思考、会不会复盘。标准答案不重要，展示你吃进去了才重要。

💼 四、项目深挖（5 分钟深度版）

⚠️ 二面面试官最可能连续追问 10 分钟 AFI 的细节。准备好"问题 → 方案 → 关键决策 → 量化效果 → 踩过的坑"五段式。
AFI 五段式（背这个）

① 问题

  • 实习期间团队每天花 ~2 小时人工刷 AI 行业信息，效率低、覆盖不全、决策层看不到提炼后的 insight
  • 核心 gap：信息过载 vs 决策需求 —— 研究员被动消费原始信息，没时间提炼

② 方案

  • 多源采集：RSS / GitHub Trending / ArXiv / HuggingFace（逐步扩展到 55+ 源）
  • LLM 相关性打分：DeepSeek V3（成本 vs 能力最优解）
  • 三层结构化输出：Raw Signal → Insight → Implication
  • 每日自动推送：GitHub Actions cron

③ 关键决策（展示 PM 思维）

  • 为什么 DeepSeek 不是 GPT-4：成本差 10x，中文能力对等，我垫 API 费能跑就行
  • 为什么三层不是一层：早期给 flat list，用户跳过太多；分层后留存率上升
  • 为什么做 topic_mode：用户反馈有两种消费模式（聚焦 vs 广谱），一套方案不能满足，做分流

④ 效果

  • 日均 10+ 频道，500+ 结构化洞察
  • 发现高价值用户（高管每周看 3-5 次）→ 针对性迭代内容
  • 覆盖 55+ RSS 源，去重准确率 95%+

⑤ 踩过的坑

  • 幻觉：LLM 编造未发生的产品发布 → 加 source citation 强制引用
  • prompt drift：调完一批数据，跑新 source 又不行 → bad case 驱动 4 轮迭代
  • 权限阻塞：内部文档 API 等 2 周没批 → 绕道用 Lobi 实现替代方案

面试官可能连追的 3 个问题

  1. "你怎么评估 AFI 的成功？" → 高管留存 + 用户反馈驱动迭代，不只看调用量
  2. "如果让你把 AFI 做成 Shopee 搜索的原型，怎么做？" → 把"信号源"换成"商品池"、"LLM 打分"换成"相关性 + 转化率多目标排序"、"分层输出"换成"分层结果页"
  3. "你遇到过什么搞不定的问题？" → prompt drift 早期我试过调 temperature / system prompt，后来发现根因是 bad case 不够系统，改成每轮 collect 20 bad cases 再迭代

📊 五、业务场景题（准备 2-3 个路径）

Q1. 产品上线后数据不如预期，怎么分析？
公式：漏斗拆解 → 分层归因 → 假设验证
  1. 先验真：数据口径变了？是否赶上特殊事件（大促后回落）？埋点 bug？
  2. 漏斗拆解：CTR 降 / CVR 降 / ARPU 降 —— 定位是哪一环
  3. 维度下钻：按用户群（新老 / 地区 / 端）、品类（生鲜 / 3C / 服饰）、时间（全天 / 周内周末）
  4. 假设验证：AB 测、用户访谈、bad case 分析三种手段
  5. 关键原则：别急着修——先诊断再动手。上线急着修是最贵的错误。
Q2. PM 和运营的区别？
公式：目标 × 手段 × 周期 三个维度
  • PM：定义 "要做什么"（what）和 "为什么"（why），产出是 PRD / 决策 / 产品形态
  • 运营：执行 "怎么让更多人用/买"（how），产出是活动 / 内容 / 增长数据
  • 周期：PM 月度 / 季度 sprint，运营每日 / 每周
  • 关系：PM 定义价值模型，运营放大价值 —— 不是上下级，是协同
  • 电商场景更交叉：Shopee 的 "AI Listing Optimizer" 既是产品功能（PM 负责）也是运营工具（运营用来指导商家）
Q3. 如何理解市场增长？
  • 三个层次：人（用户规模）× 次（频次）× 钱（客单价） = GMV
  • 东南亚电商：新用户增长（从 0-1）+ 老用户复购频次（1-10）+ 消费升级（客单价提升）
  • Shopee 现在处在：头部市场（印尼）接近饱和 → 泰国马来增长 → 新兴市场（菲律宾 / 越南）拓展
  • AI 在每一层都有作用：个性化找新用户 / 推荐提频 / AI 导购提升客单
Q4. 你通过客户需求怎么指导项目执行？
  • 结构化需求收集：不是 "what"，是 "why behind what"（探 underlying 场景）
  • 优先级框架：影响面 × 紧急度 × 实现成本（RICE / ICE 都可以讲）
  • MVP 思维：先做最小可验证版本跑数据，不是等完美方案
  • AFI 例子：高管说"想看前沿动态"，我没做 100 源大爬虫，先做 5 源看留存；留存好再扩源
