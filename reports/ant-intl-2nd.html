情报 2：东南亚 AI 搜索的三个特有约束（极重要）

除了语言多样性（英/印尼/泰/越/马来/繁中），面试官透露还有三个关键约束：

约束 1：东南亚语种大模型能力弱

  • 印尼、泰、越的训练语料比中英少一个量级
  • 直接用 GPT-4 / Claude 效果掉一档
  • 产品含义：不能假设"上 LLM 就完事"，要 fine-tune 或蒸馏本地化模型；benchmark 要自建

约束 2：用户 AI 素养普遍较低

  • 没被 ChatGPT / 豆包教育过
  • 不会"精心组织 prompt"，也不理解"AI 会出错"
  • 产品含义：交互要更 hand-hold（意图澄清 chip 比搜索框重要）；教育成本高，但谁先教育谁先占位

约束 3：Query 质量普遍偏低

  • 拼写错、语法乱、极短极模糊、跨语言混用
  • 产品含义：LLM 改写 query 在东南亚 ROI 是国内的好几倍；零结果率高 → 要强兜底
综合推论：东南亚是"低垂果实"——谁先解决"本地语言 × 低 AI 素养 × 低 query 质量"的组合问题，谁先占位。这三个约束在蚂蚁国际的支付 / 理财 / 跨境汇款场景同样成立，甚至更严重（金融术语 + 合规压力 + 用户信任成本高）。

🎁 五、四层 Landscape 深化（每层 3 条产品判断 + 蚂蚁国际的机会）

Layer 1：MaaS 中台（对应蚂蚁百灵）

  1. 东南亚语种模型 fine-tune = 差异化护城河。阿里云通义、字节火山方舟都在做通用大模型，但没人深做东南亚。蚂蚁国际深耕东南亚金融 → 语料积累有优势。
  2. 金融垂域微调：通用 LLM 对金融合规、风控、反洗钱等场景能力弱 → 蚂蚁百灵可以做金融场景的行业大模型。
  3. 关键指标：API 调用量、跨 BU 复用率、东南亚语种任务的 MMLU / benchmark 得分（自建）。

Layer 2：类 OpenClaude（研发 / 办公提效）

  1. 跨语言代码注释 + code review：蚂蚁国际研发团队跨国，代码注释里有中英印尼越多语混用——这是普通 OpenClaude 搞不定的特殊场景。
  2. 合规 code review：金融代码对风控规则、合规接口调用有强要求，AI 可以做静态合规检查。
  3. 给非技术员工：运营、商家服务、客服的 Copilot——对应 Shopee 的第二层。

Layer 3：类 Dify（AI 应用搭建平台 · for 内部）

  1. 多语言配置模板：让各 BU 搭 AI 应用时，语言 / 合规模板一键切换，不用每国重新开发。
  2. 跨 BU 能力复用：平台沉淀 "风控检查"、"身份核验"、"汇率换算" 等金融原子能力，AI 应用直接调用。
  3. 关键指标：平台上搭建的应用数 / 跨 BU 复用率 / 非技术人员自建应用比例。

Layer 4：toC 产品（Alipay+ · 跨境汇款 · 理财顾问 · AI 客服）

  1. 多模态入口：东南亚用户打字门槛高（语言多 + 拼写差），语音搜 + 拍立搜 ROI 比搜索框高得多。
  2. 意图澄清优先：query 质量低 → 不能直接搜，要多轮对话澄清（尤其金融场景，搞错代价大）。
  3. Context 积累：用户在 Alipay+ 的历史 transaction / 偏好 / 汇款目的都是独家资产——Agent 在这里表现最好，迁移成本最高。

💎 六、可以主动抛的三个亮点观点

亮点 1：搜推从前台产品退到后台引擎（来自阿里 Q25）

"现在的搜推靠 UI 设计、标题党、图片来影响用户——这些是为'人类注意力'设计的。但如果用户的 Agent 替他们看，Agent 不吃标题党，它看的是结构化数据。平台搜推会从'给人看'变成'给 Agent 调用'。"

亮点 2：MCP 让平台变成 Agent 数据入口（来自阿里 Q33）

"MCP 让 Agent 标准化连接外部服务，类比 USB-C。用户 Agent 可以同时调蚂蚁、Shopee、Lazada 接口做跨平台比价、跨境汇款。平台竞争会从 app 体验变成 MCP 接口开放度 + API 稳定性。Shopee 已经在做了，蚂蚁国际如果动作慢，会错过 Agent 时代的入口位置。"

亮点 3：东南亚是"低垂果实"（来自今早 Shopee 情报）

"东南亚 AI 搜索有三个特有约束——模型能力弱（语料少）、用户 AI 素养低、query 质量差。这三个约束在蚂蚁国际的金融场景更严重。但反过来——谁先解决这三个约束的组合问题，谁就占位了整个东南亚 AI 金融的基础设施层。蚂蚁国际深耕东南亚十年，数据资产和关系网络都有优势，是最有条件吃这块果的玩家。"

  4. 组织形态："国际 AI 团队是中心化（一个大团队服务所有 BU）还是嵌入式（每个 BU 自己的 AI 小组）？"
