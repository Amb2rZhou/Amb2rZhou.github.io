💼 四、Mock 12 题（按出现概率排序）

A. 业务理解 + 战略洞察（必问 3-4 题）

Q1. 你怎么看 TikTok 在美国的现状？🔥
公式：现状（数据） → 关键 risk → 机会 → 你的判断
  • 现状："2026 年 1 月美国业务重组完成，字节保留 19.9% 股权 + 算法 ownership，但数据/审核合资化"
  • Risk：内容审核收紧 → 创作者生态可能流失 → Facebook Reels 趁机挖人
  • 机会：商业化（电商 / 广告）仍全资 → TikTok Shop 已经做到 158 亿美金 → 这才是真增长曲线
  • 判断："美国业务的核心竞争从 user growth 转向 monetization quality。Reels 抢用户但抢不走 TikTok Shop 的电商闭环"
Q2. TikTok 在东南亚怎么打 Shopee？
公式：差异化优势 → 协同模式 → 长期格局
  • TikTok 优势：社交流量 + 内容驱动发现 + 创作者生态
  • Shopee 优势：货架式电商 + 物流基础设施 + 商家关系
  • 战略："不是 zero-sum。TikTok 在 social discovery + impulse buying，Shopee 在 search-driven + considered purchase。两者协同存在 → 但 TikTok Shop 已经做了仓储自营，可能会侵蚀"
  • 东南亚特殊性：印尼 / 泰国 / 越南消费者短视频渗透率极高 → TikTok Shop 增长比美国更快
⚠️ 你刚拿 Shopee offer，不要 over-praise TikTok 抨击 Shopee。中性分析显示成熟度。
Q3. TikTok 当前最大的战略 challenge 是什么？
  • 短期：美国监管不确定 + 内容审核收紧
  • 中期：用户增长放缓 → 必须靠商业化（Shop / 广告）维持估值
  • 长期：算法核心 IP 一旦被监管化或开放，差异化壁垒会被稀释
  • 你的判断："长期看，TikTok 必须从'算法平台'升级到'商业基础设施'，让创作者 + 商家在这里赚钱比在别处多 → 这才是 anti-Reels 的真正护城河"

B. 数据 / 行业研究方法（必问 2-3 题）

Q4. 怎么评估一个新功能上线对 TikTok 的影响？
公式：北极星指标 + 漏斗 + AB + 长期效应
  • 北极星：DAU × engagement time（不是单一 metric）
  • 漏斗指标：曝光率 / 点击率 / 完播率 / 互动率（赞、评、分享）/ retention
  • AB 测试：先 5% 流量 → 看核心 metric 不变差才放大
  • 长期 watch：内容生态健康度（创作者投稿量 / 多样性）+ 商业化（GMV / 广告 CTR）
  • 避坑：短期 metric up 但 retention down = 假信号
Q5. 给你一个开放问题：TikTok Shop 在马来西亚的 GMV 上个月下降了 15%，怎么分析？
公式：四步拆 (验证 → 公式 → 维度 → 假设)
  1. 验真：数据口径变了？大促后回落？埋点 bug？
  2. 公式拆：GMV = 买家数 × 转化率 × 客单价 → 哪个因子降
  3. 维度下钻：品类（生鲜/服饰/3C）/ 用户群（新/老）/ 地区（KL vs 二线）/ 时间（全月 vs 某天异常）
  4. 假设：竞对补贴？算法调整？合规变化？汇率？季节性？
回答时主动说："我的思路是先验真，沿公式拆，再下钻看是全局还是局部。您希望我从哪个方向深挖？"——把球抛回去显示框架感
Q6. 你怎么用 SQL 看 TikTok 的用户活跃度？
公式：诚实 + 简单查询 + 框架
  • 诚实："我熟悉 SQL 基础查询（join / group by / window function），但工业级海量数据的 SQL 优化我不是专家。"
  • 简单写法： 
    SELECT
  date_trunc('day', event_time) as day,
  count(distinct user_id) as DAU,
  count(*) / count(distinct user_id) as avg_sessions
FROM tiktok_events
WHERE event_type = 'video_view' AND date >= '2026-04-01'
GROUP BY 1 ORDER BY 1;
  • 升级版：用 window function 算 retention（user 在 day 0 后 day 1/7/30 是否回来）
  • 关键：DAU 是表层，retention 是更核心的

C. 项目深挖 AFI（必问）

Q7. AFI 具体怎么做的？解决了什么问题？
五段式：问题 → 方案 → 关键决策 → 量化效果 → 坑
  1. 问题：实习期间团队每天花 ~2 小时人工刷 AI 行业信息，效率低、覆盖不全、决策层看不到提炼后的 insight
  2. 方案：多源采集（55+ RSS / GitHub / ArXiv / HuggingFace）→ LLM 相关性打分 (DeepSeek V3) → 三层结构化输出（Raw Signal → Insight → Implication）→ 每日自动推送
  3. 关键决策：
    • 选 DeepSeek V3 而不是 GPT-4：成本差 10x，中文够用
    • 三层而非 flat list：用户能秒定位需要的层级
    • topic_mode 分流：分析师 vs 决策者两类用户消费模式不同
  4. 效果：日均覆盖 10+ 频道，500+ 结构化洞察，发现高价值用户 pattern → 推动内容针对性迭代
  5. 坑：幻觉（加 source citation 解决）/ prompt drift（4 轮 bad-case 驱动迭代）/ 文档 API 权限阻塞（绕道用 Lobi 实现替代方案）
Q8. AFI 跟 TikTok 战略生的工作有什么关系？

挂钩话术：

"AFI 本质就是战略生工作的 prototype——'多源数据 + LLM 提取 + 给决策层结构化洞察'是同一套方法论。

不同：
① 标的从 'AI 行业趋势' 换到 'TikTok 国际化业务'
② 数据源从 ArXiv / GitHub 换到 SQL / AppAnnie / Questmobile
③ 输出从 daily insight 换到 quarterly strategy report

但 pipeline 完全可迁移。我做 AFI 时踩过的坑（信号筛选 / 假信号 / 用户分层）在 TikTok 战略报告里同样会遇到，我有体感。"
