🎯 一、为什么 IDG 不同于互联网大厂（先想清楚这个）

核心认知：IDG 是投资机构，不是产品公司。"AI Solutions PM" 在投资机构的含义和在大厂完全不同。

互联网大厂 PM vs 投资机构 PM 的 5 个差异

维度互联网大厂 PM投资机构 "AI Solutions PM"
产品面向C 端用户或商家内部投资团队 / LP / portfolio companies
成功指标DAU / GMV / 留存投资决策质量 / DD 效率 / portfolio 价值
迭代节奏周级 / 月级 sprint季度级 / 年级（贴合投资周期）
团队规模几十到几百人几人到十几人
核心能力用户洞察 + 执行力行业判断 + 跨界整合 + 研究深度
面试重点：IDG 想要的不是一个"会做 C 端产品的 PM"，是一个懂 AI + 懂商业 + 能独立研究 + 能快速学习行业的人。你的 AFI 项目直接命中这个画像。

🧭 二、AI Solutions PM 在 IDG 到底做什么（4 种可能）

可能 1：对内 —— 投资决策提效工具

  • 给 GP / 分析师做 deal sourcing、行业研究、尽调工具
  • 类似"内部 Copilot for investors"
  • 你的挂钩：AFI（多源信息召回 + 分层洞察）本身就是这个场景的雏形

可能 2：对外 —— 赋能 portfolio companies

  • 帮被投公司做 AI 转型咨询 / AI 能力接入 / 最佳实践沉淀
  • 类似"AI consulting + PM"混合角色
  • 你的挂钩：intern-clawd 作为 Copilot + wiki 的完整实现，是可演示的 case

可能 3：LP 侧 —— 报告与智能洞察

  • 给 LP 定制化报告、ESG 分析、组合表现智能摘要
  • 你的挂钩：AFI 生成日报/周报的 pipeline 直接复用

可能 4：IDG 自己孵化 AI 产品

  • IDG 可能在内部孵化某个 AI SaaS 工具（给金融/法律/咨询行业）
  • 这个岗位是帮忙把某个 AI 产品从 0 到 1 做出来

💼 五、Mock 12 题（IDG 专属）

A. 动机 / 定位（3 题）

Q1. 为什么选择投资机构的 AI Solutions，而不是互联网大厂的 AI PM？
公式：高密度专业人士场景 + 跨行业视野 + 0-1 机会
  • "互联网大厂的 AI PM 是优化已有产品——给 1 亿用户的搜索加 AI。这个很有价值，但边际空间在缩小。"
  • "投资机构的 AI Solutions 是服务高密度专业人士——每个分析师每天处理的信息量是普通用户的 100 倍，AI 能提效的空间更大，但也更难。"
  • "第二点：投资机构有跨行业视野——一个月能看遍 10 个行业的前沿。这对我的长期成长比聚焦单一业务更重要。"
  • "第三点：投资机构的 AI 能力还没被标准化，很多是 0-1 阶段——这种阶段的 PM 能真正定义问题，而不是填补空位。"
Q2. 你怎么看投资机构 AI 化的节奏？我们是不是在追热点？
公式：分层回答 + 不卑不亢 + 显示自己有判断
  • "短期（1 年）：AI 在投资机构的落地有一些'追热点'的成分——每个基金都在招 AI 产品经理，但很多还没想清楚具体做什么。"
  • "中期（2-3 年）：分化会开始——真正想清楚的机构会用 AI 建立信息处理的结构性优势，想不清楚的会浪费资源。"
  • "长期（5 年）：AI 能力会成为基础设施，不再是差异化——但在这 2-3 年的差异化窗口期，先动的机构会获得决定性优势。"
  • 加分："所以我觉得 IDG 现在招这个岗位，时机是对的——不是最早但也不晚。关键是 2-3 年内能不能跑出差异化的东西。"
Q3. 实习生能做什么？你期望能产出什么？
  • "实习生最大的价值是快速验证 + 快速试错——不承担长期责任，但能快速跑 pilot。"
  • "我期望能做两件事：
    ① 验证一个具体的 AI 工具假设（比如某个内部研究 Copilot），3 个月内出 MVP + 用户反馈；
    ② 沉淀一套可复用的方法论（prompt 工程规范 / 评估框架 / 数据管道），实习结束后团队能继续用。"
  • "我不期望入职三个月成为专家，但期望能让团队在 3 个月后比我来之前节省 X% 的时间在信息处理上。"

B. AFI 深挖（4 题 —— 这是你的主场，准备充分）

Q4. AFI 具体怎么做的？（5 分钟详细版）
公式：问题 → 方案 → 关键决策 → 量化效果 → 坑
  1. 问题：团队每天花 2 小时刷 AI 行业信息，效率低、覆盖不全、决策层看不到提炼后的 insight
  2. 方案：多源采集（RSS / GitHub / ArXiv / HuggingFace / X/Twitter）→ LLM 相关性打分 → 三层结构化输出（Signal / Insight / Implication）→ 每日自动推送
  3. 关键决策：
    • 选 DeepSeek 而不是 GPT-4：成本 + 中文能力足够
    • 选 signal/insight/implication 三层而不是 flat list：用户能快速定位到决策需要的层级
    • 做 topic_mode 分流："聚焦型"和"广谱型"用户消费模式不同
  4. 效果：日均覆盖 10+ 频道，积累 500+ 条结构化洞察，发现内部高价值用户 pattern
  5. 踩过的坑：
    • 幻觉（LLM 编造未发生的事件）→ 加了 source citation 强制
    • prompt drift（4 轮调优）→ bad case 驱动
    • webhook 文本推送无法追踪阅读 → 改成文档形式，原生浏览记录
Q5. AFI 和投资机构的 deal sourcing / 行业研究有什么关系？
公式：三层同构 + 具体场景 + 差异说明

三层同构：

AFI 对技术决策者投资机构对投资决策者
多源采集deal sourcing（多源 pipeline）
相关性打分初筛（匹配基金方向）
三层输出行业地图 + 核心公司 + 投资建议

差异：

  • AFI 的用户是工程师，容忍一定错误率；投资决策容错更低
  • AFI 输出是通用 signal；投资研究要individual company 深度
  • AFI 完全自动化；投资决策一定要 human-in-the-loop，AI 是 augment 不是 replace
Q6. 如果让你在 IDG 做一个类似 AFI 的工具，第一版怎么设计？
公式：选一个具体场景 → MVP → 扩展
  • 选场景：AI/科技赛道的 deal pipeline dashboard（从 YC / ProductHunt / GitHub trending / 行业推文抓最新创业公司，按基金方向打分）
  • MVP：
    • Week 1-2：数据源接入（8-10 个 pipeline）
    • Week 3-4：打分 prompt + 分析师反馈回路（analyst 标 good/bad → 改 prompt）
    • Week 5-8：dashboard + 日报自动化
  • 扩展：
    • 竞争情报（持续跟踪已投公司的竞对动态）
    • LP 定制化报告自动生成
    • portfolio company 的内部 AI 工具咨询
Q7. AFI 做了这么久，为什么还是你一个人？是不是不够重要？
  • "这是个好问题。坦诚说，我从一开始就是个人项目——内部产品，目标是先用 MVP 证明价值，再申请更多资源。"
  • "我已经做到的：高管用户 engagement 数据在上升，我每周都能收到他们的 feedback 和新需求。"
  • "接下来要做的：基于数据去申请正式立项 + 工程师资源。这也是我来 IDG 想学的——如何在投资机构这种资源相对紧的环境下，用 MVP 说服资源倾斜。"
  • 加分："而且一个人做的好处是：从前端到后端我都踩过坑，能非常快判断哪些需求是真实的、哪些是功能虚荣。"

C. 投资机构场景题（3 题）

Q8. 投资团队用 AI 最大的阻力是什么？
公式：不是技术，是信任 + 工作流 + 怕被替代
  • 信任：投资决策涉及真金白银，LLM 幻觉风险无法接受。解法：AI 只做信息汇总和 flag，决策权在人。
  • 工作流：分析师已有的研究流程（Excel / Notion / 邮件）高度个人化，硬塞工具会被拒。解法：嵌入现有工作流，不做新 app。
  • 怕被替代："AI 会不会取代我"的隐性焦虑。解法：把 AI 定位成 augment 不是 replace，让分析师看到 AI 让他们"更像合伙人"而不是"可被替代"。
"三个阻力里，工作流嵌入是最容易被低估的——很多 AI 工具技术没问题但没人用，就是因为不贴合既有习惯。这是我在 intern-clawd 里反复踩的坑——所以做了 CLI、快捷键、IM、webhook 5 个入口，就是为了适应不同场景。"
Q9. 如果让你评估一个"AI + 金融"的 startup，你会看什么？
  • 独有数据：有没有别人拿不到的数据源？（比如对接交易所 API / 行业独家数据）
  • 目标用户密度：是服务高密度专业人士还是大众？前者单价高粘性强，后者流量红利易逝
  • 模型/工具护城河：是调用通用 LLM 还是自己有 fine-tune 能力？前者容易被复制
  • 合规护城河：有没有金融牌照 / 合规能力 / 关系网络？在强监管行业这是护城河
  • 商业模式：SaaS / API / 交易分成 / 顾问服务 —— 不同模式估值逻辑完全不同
Q10. IDG 投了很多 AI 公司，你最看好哪个方向？为什么？
公式：选一个 + 3 条理由 + 承认风险

建议选"企业级 AI Agent / 内部 Copilot"方向（因为你自己做 intern-clawd，讲得透）

  • 理由：
    1. C 端 AI 应用的流量红利已经被 ChatGPT 这类吃完，差异化难
    2. 企业级 Agent 的门槛是"理解专业场景 + 嵌入工作流"，不是"模型多强"——更符合 startup 的 lean 打法
    3. 企业客户付费能力强，不烧补贴
  • 承认风险："但企业级 Agent 的坑是 sales cycle 长、客户获取慢，对现金流挑战大。所以看 startup 的关键是 product-led growth 能力——能不能让用户自传播。"
  • 挂钩："这也是我做 intern-clawd 的思路——先做个人版，通过开源自传播，再往团队/企业延展。"

D. 反转 / 压力题（2 题）

Q11. 你的项目听起来很技术，但 IDG 要的是商业判断，你怎么证明你有商业思维？
  • "这个反问很公平。我先承认一点：我的技术执行力比商业判断更强——前者有 AFI / intern-clawd 可证，后者是我现在要学的。"
  • "但我不是没有商业思考：
  • intern-clawd 的发布策略：我做了 3 层营销叙事分层投放——极客圈（LLM Wiki 论述）、Claude 用户（把订阅变成 OpenClaude）、普世大众（Context 红利）。这是对用户心智和传播路径的判断。
  • Context 红利的论述：我写过一篇分析'AI 是 Gen Z 的时代红利，Context 是终极护城河'——这是对行业变化节奏的判断，不是技术讨论。
  • "所以我的'商业判断'目前停留在技术驱动的商业直觉层面——我清楚技术会怎么演化、哪些变化会产生商业机会。但对 CAC / LTV / 交易结构这些金融层面的判断，我承认我还需要学习。这也是我想来 IDG 的原因——在这里一个月能学到的商业思维，在大厂要一年。"
Q12. 如果有人给你另一个 offer——阿里的 AI PM、蚂蚁的 AI PM、或字节的 AI PM——你为什么还要来 IDG？
  • "坦诚说，大厂 offer 和投资机构 offer 解决的是不同阶段的问题。"
  • "大厂 AI PM 的价值：快速见到大规模产品效果，一个功能影响上亿用户。职业早期建立'做过大产品'的背书。"
  • "投资机构的价值：跨行业视野 + 高密度学习 + 0-1 定义问题的锻炼。职业早期建立'能独立思考'的能力。"
  • "对我自己的判断：我已经具备'做产品'的经验（小红书实习 + AFI + intern-clawd），现在需要补的是'横向视野'——看更多行业、更多商业模式。这是大厂给不了的。"
  • "但如果 IDG 问我要不要 full-time commitment，我现在不能承诺——实习三个月是我评估这个方向的方式。如果三个月后觉得投资 + AI 是我想长期做的，我会主动争取留下。"
核心原则：不承诺全职、但诚实说出自己的判断框架。面试官想要的是会思考的人，不是盲目忠诚的人。
